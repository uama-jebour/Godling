# 技术架构摘要

## 顶层状态

### `ContentDB`

- 负责加载 JSON 内容定义
- 负责 `id` 查询、标签检索和引用校验

### `ProgressionState`

- 负责跨局永久状态
- 包含永久仓库、永久货币、已完成任务、剧情标记、系统解锁
- 包含家园配置（英雄选择、携带物、装备快照来源）

### `RunState`

- 负责当前单局状态
- 包含地图、回合、危险度、事件板、临时战利品、局内地图数值和局内完成任务

## 模块分层

### 世界层

- 事件板生成
- 任务池与固定事件线
- 回合推进
- 事件结算分发
- 强制非回合事件触发（`forced_non_turn`）

### 战斗层

- `battle` 类型事件的执行器
- 单位、AI、战斗内事件、胜负结果

### 进度层

- 撤离结算
- 死亡清算
- 永久存档

### UI 层

- 展示状态
- 接收输入
- 不直连底层状态细节

## 推荐目录

```text
autoload/
data/
systems/
ui/
scenes/
tests/
```

文档目录则固定为：

```text
docs/
features/
tasks/
```

## 接口约束

- 战斗层向世界层回传 `BattleResult`
- 世界层向进度层回传 `ExtractionResult`
- 任何 UI 不应直接决定奖励或剧情写入
- 内容来源优先配置，不优先脚本硬编码
