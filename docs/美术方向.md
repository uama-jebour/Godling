# 美术方向

## 当前阶段定位

MVP 阶段的美术目标不是定稿，而是保证玩法信息可读、资源命名稳定、后续易替换。

## 视觉约束

- 游戏是 2D 俯视或近俯视表达
- 地图上的随机事件与固定事件必须一眼可分
- 战斗中的敌我阵营必须高可读
- 藏品、物资、货币图标在列表中要能快速区分

## 事件板可读性

- 固定事件与随机事件不要用完全相同的视觉语言
- 固定事件应更稳定、更像长期挂在地图上的目标点
- 随机事件更像当回合临时出现的机会点

## 占位资源规则

- 占位图允许使用简单几何图形或统一风格图标
- 占位资源命名必须稳定，后续可无痛替换
- 不要在脚本中写死某张占位图的特殊逻辑

## 当前不要过早做的事

- 不要在没有玩法定型前做重美术投入
- 不要让 UI 风格先于系统信息架构确定
