# 内容管线说明

## 来源与落地

内容生产流程固定为：

`Excel 策划源 -> 手工整理或脚本导出 -> JSON -> ContentDB 校验 -> 运行时加载`

当前 Excel 来源：

- `战斗系统.xlsx`
- `大地图系统设计.xlsx`
- `物品表.xlsx`

## 运行时内容文件建议

- `items.json`
- `loot_tables.json`
- `maps.json`
- `events.json`
- `tasks.json`
- `units.json`
- `battles.json`
- `battle_events.json`
- `progression_defaults.json`

## 内容命名规则

### 文档文件

- 使用中文文件名，便于理解

### 运行时资源与代码文件

- 使用 ASCII
- 推荐 `snake_case`
- `id` 使用稳定英文标识，不使用中文

### 配置字段

- `spawn_mode` 与 `resolution_type` 必须拆开
- 不允许只用单个 `event_type` 模糊表达两类信息
- 强制事件使用 `trigger_mode = forced_non_turn`

## 最低校验要求

加载 JSON 时必须校验：

- 重复 `id`
- 缺失必要字段
- 任务引用不存在事件
- 事件引用不存在战斗
- 掉落池引用不存在物品
- 地图随机锚点数量不足

## 当前阶段原则

- 先做最小内容子集，不追求一次性把三张表完整导完
- 每个系统先选 1 组可跑通样本内容
- 当 schema 不稳定时，先修契约，再扩大内容量
- 首批命名统一为：`world_a` + `map_world_a_02_*`（A世界第2张地图）
