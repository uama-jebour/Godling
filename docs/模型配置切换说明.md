# Codex 模型配置切换说明

本文档说明如何在已有的多个 Codex 配置文件之间快速切换。

## 1. 配置文件位置

仓库内同步文件目录：

- `codex-configs/`

仓库内当前已维护的配置文件：

- `codex-configs/config_codex_5.3.toml`（当前配置备份）
- `codex-configs/config_glm-5.toml`
- `codex-configs/config_kimi-k2.5.toml`
- `codex-configs/config_minmax-m2.5.toml`
- `codex-configs/config_official_gpt_login.toml`（官方 GPT 登录链路）

本机生效目录：

- `~/.codex/`

本机目标文件：

- `~/.codex/config.toml`（Codex 实际读取）

## 2. 跨机器同步（推荐）

在新机器（例如家里电脑）拉取仓库后，先执行：

```bash
./scripts/install_codex_configs.sh
```

这会把 `codex-configs/` 下的配置文件同步到 `~/.codex/`。

如果希望同步后立刻激活某个配置：

```bash
./scripts/install_codex_configs.sh --activate codex-5.3
./scripts/install_codex_configs.sh --activate glm-5
./scripts/install_codex_configs.sh --activate kimi-k2.5
./scripts/install_codex_configs.sh --activate minmax-m2.5
./scripts/install_codex_configs.sh --activate official-gpt
```

## 3. 一键脚本切换（推荐）

项目内已提供切换脚本：

- `scripts/switch_codex_model.sh`

先看可切换目标：

```bash
./scripts/switch_codex_model.sh --list
```

切换示例：

```bash
./scripts/switch_codex_model.sh codex-5.3
./scripts/switch_codex_model.sh glm-5
./scripts/switch_codex_model.sh kimi-k2.5
./scripts/switch_codex_model.sh minmax-m2.5
./scripts/switch_codex_model.sh official-gpt
```

脚本行为：

- 自动把当前 `~/.codex/config.toml` 备份为 `config.toml.switch-backup.<时间戳>`
- 再把目标配置复制为新的 `~/.codex/config.toml`
- 输出当前生效的 `model` / `model_provider`

## 4. 手动切换步骤（通用）

在终端执行下面任一命令，把目标配置覆盖到 `config.toml`：

```bash
cp ~/.codex/config_glm-5.toml ~/.codex/config.toml
```

```bash
cp ~/.codex/config_kimi-k2.5.toml ~/.codex/config.toml
```

```bash
cp ~/.codex/config_minmax-m2.5.toml ~/.codex/config.toml
```

```bash
cp ~/.codex/config_official_gpt_login.toml ~/.codex/config.toml
```

## 5. 切换后检查

建议每次切换后检查生效内容：

```bash
cat ~/.codex/config.toml
```

重点确认：

- `model = "..."` 是否是目标模型
- 若使用聚合通道模型（glm/kimi/minmax），应包含 `model_provider = "my-newapi"`
- 若使用官方 GPT 登录配置，通常不需要 `model_provider = "my-newapi"`

## 6. 恢复到原始 Codex 5.3 备份

```bash
cp ~/.codex/config_codex_5.3.toml ~/.codex/config.toml
```

## 7. 生效时机

建议在切换后重新打开 Codex 会话（或重启相关终端会话），避免旧会话继续沿用之前的配置上下文。

## 8. 常见问题

1. 切换后模型没变
- 检查是否覆盖到了 `~/.codex/config.toml`（不是项目里的 `.codex/`）。
- 重新开启新会话再验证。

2. 官方 GPT 登录配置报认证问题
- 先确认你本地官方登录状态有效（按你当前使用方式重新登录一次）。

3. 想新增一个新模型配置
- 复制任意一个现有配置文件，修改 `model = "新模型名"` 后保存为新文件，先按第 2 节同步，再按第 3 节切换。
