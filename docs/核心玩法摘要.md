# 核心玩法摘要

## 单局主循环

单局流程固定为：

1. 在家园界面配置英雄与携带物品，并选择目标地图
2. 进入地图后读取局外进度并初始化 `RunState`
3. 生成当回合事件板
4. 玩家选择并处理 `1` 个事件
5. 结算奖励、损失、任务推进与地图数值变化
6. 回合推进、危险度上升
7. 玩家决定继续或撤离
8. 若选择撤离，触发撤离事件并结算回家

## 事件板规则

### 随机事件

- 每回合生成大约 `4-6` 个
- 来自随机池
- 刷新在地图固定位置上
- 若未被选择，下回合可能被刷新走

### 固定事件

- 数量不定
- 来自任务线、叙事线、key 物品、保底逻辑或特殊条件
- 不占随机事件槽位
- 默认跨回合保留

### 玩家选择规则

- 玩家每回合只处理 `1` 个事件
- 其他随机事件在回合结束时可重刷
- 固定事件在被解决、失效或替换前持续存在
- 标准样例中第一回合为 `4` 个随机 + `1` 个固定主线事件，总计 `5` 个
- 标准样例中第二回合为 `5` 个随机 + `1` 个固定主线事件，总计 `6` 个

## 事件的两条轴

### 出现方式 `spawn_mode`

- `random_slot`
- `fixed_line`

### 处理方式 `resolution_type`

- `battle`
- `random`
- `narrative`

推荐理解：

- 大多数叙事事件：`fixed_line + narrative`
- 常规战斗事件：`random_slot + battle`
- 资源探索或轻量遭遇：`random_slot + random`
- 特殊剧情战斗：`fixed_line + battle`
- 强制袭击事件：`fixed_line + battle + forced_non_turn`

## 强制事件与撤离事件

- 第四回合起可以按概率触发强制恶魔袭击事件。
- 强制事件不占回合选择次数，属于额外处理。
- 主线完成后撤离会触发撤离事件，撤离成功后才可带出战利品。

## 收益与失败

### 局内临时收益

- 本局获得的物品
- 本局获得的货币
- 本局获得的藏品

### 局外永久收益

- 撤离后正式带出的物资和货币
- 已完成任务
- `story_flags`
- `unlock_flags`

### 死亡规则

- 未撤离的临时物资、货币、藏品丢失
- 剧情标记和系统解锁默认保留

## MVP 不做什么

- 抛射和墙体破坏
- 复杂主动施法
- 掉落池嵌套
- 家园和图鉴
- 撤离战
