# 当前任务卡

## 当前阶段目标

当前默认优先任务是：

`巩固最小可玩闭环，并补齐任务骨架可视化 / 交互战斗表现 / 编辑器与发布链路`

## Goal

完成“首屏地图任务骨架可读 + 进入交互战斗 + 结果回写 + 可在 Web 试玩 + 可在运行时调参与管理内容”的稳定原型。

## Scope

只做：

- 首屏地图事件板与右侧事件面板联动
- 任务 / 主线节点可视化与叙事选项文本级展示
- 交互战斗的基础操作、结算与回写
- 战斗结果面板、技能位与攻击演出手感优化
- 首屏布局与窗口适配
- 数值编辑台与全内容库编辑器
- Web 导出预设与 GitHub Pages 发布链路
- 文档与交接同步

不做：

- 完整职业/技能树
- 完整背包拖拽 UI
- 复杂 AI 与高级战斗特效
- 最终美术资源替换
- 完整剧情分支演出
- 大规模内容量产

## 验收标准

- 项目启动后首屏能显示地图事件板
- hover 地图事件点时，右侧事件面板实时联动
- 右侧可看出“当前任务 / 主线节点 / 下一节点提示”
- 叙事事件支持 1-3 个选项的结构化预览与结果展示
- 点击战斗事件后可进入交互战斗并正常回写结果
- 战斗结束后会先出现战果卡，再返回地图
- 普通流程、强制袭击、撤离流程都保持可用
- 数值编辑台能分类展示并应用主要平衡参数
- 全内容库编辑器能对内置/自定义英雄、敌人、道具做增删改查
- 内容引用编辑支持 `敌人 -> battle`、`道具 -> loot table`、`loot table -> 事件奖励`
- 本地 `Web` 导出命令可成功产出浏览器构建
- GitHub Pages 页面可直接访问
