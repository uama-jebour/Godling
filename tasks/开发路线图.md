# 开发路线图

## 阶段 0：工程底座

目标：

- 建立 Godot 工程结构
- 建立 3 个顶层状态
- 建立 JSON 加载和校验

完成标准：

- 项目可启动
- JSON 能加载
- 坏引用可报错

## 阶段 1：世界地图与事件板

目标：

- 实现随机事件与固定事件并存的回合事件板
- 支持每回合 `4-6` 个随机事件
- 支持固定事件跨回合保留
- 支持第4回合后的概率强制袭击（不占回合）

完成标准：

- 玩家可进入地图并连续推进多个回合
- 每回合可处理 1 个事件

## 阶段 2：任务与叙事

目标：

- 用任务驱动固定事件线和叙事推进
- 支持 `story_flags` 和 `unlock_flags`

完成标准：

- 完成任务后可出现后续固定事件或系统入口

## 阶段 3：战斗执行器

目标：

- 拉起 `battle` 类型事件
- 实现基础自动战斗与战斗内事件

完成标准：

- 世界层可进入战斗并接收 `BattleResult`

## 阶段 4：背包、撤离与进度

目标：

- 建立临时战利品
- 建立撤离转存和死亡清算
- 建立装备和永久仓库
- 建立家园配置（英雄与携带物）到单局开局的连接
- 建立撤离事件（可为战斗）到回家结算的连接

完成标准：

- 单局可完整进入、推进、撤离并保留成果
