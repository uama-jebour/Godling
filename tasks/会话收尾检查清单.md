# 会话收尾检查清单

每次结束开发前，按顺序勾选：

- [ ] `tasks/开发交接.md` 已更新（今日完成、下一步、风险、关键文件）。
- [ ] `项目记忆.md` 已同步关键决策与已知坑点。
- [ ] `tasks/当前任务卡.md` 已收敛到“下一次只做什么”。
- [ ] 本次新增脚本/命令已写入 `README.md`（如有）。
- [ ] 至少执行 1 次可复现验证（手动或自动）并记录结果。
- [ ] 本地可运行（Godot 可启动、主流程无阻塞报错）。
- [ ] 本次改动已提交（小步 commit，信息包含意图）。
- [ ] 需要跨设备继续时，已 push 到远端分支。
- [ ] 若达到可演示节点，已创建或更新 Release。
- [ ] 下次会话启动提示词已准备好（可直接复制下面模板）。

## 下次会话启动模板（复制即用）

请先阅读以下文件并仅加载必要上下文：

1. `agent.md`
2. `项目记忆.md`
3. `tasks/当前任务卡.md`
4. `<本次功能相关 spec 文件>`
5. `tasks/开发交接.md`

在动手前先输出 5 条：

1. 你理解的目标  
2. 你不会做的内容  
3. 你会改哪些文件  
4. 你要遵守的限制  
5. 你如何验证
