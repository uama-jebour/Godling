## 风险与阻塞

- 当前阻塞：
  - 无硬阻塞；当前主要是体验与编辑器效率问题
- 风险点：
  - 如果换机器后只同步仓库、不安装 skill 到 `~/.codex/skills/azure-imagegen`，则无法直接通过 skill 名称触发 Azure 出图工作流
  - 如果新机器未写入 4 个 Azure 环境变量，`azure_image_gen.py` 会直接报缺失变量
  - Godot 编辑器内试玩窗口行为与项目运行时窗口控制并不完全一致
  - Web 版可能还会暴露字体、音频、缩放、持久化等浏览器侧差异
  - 引用挂载已可用，但若缺少“引用关系总览 + 解绑”会导致迭代期排错成本偏高
- 需要决策：
  - 后续优先补“引用关系总览 / 解绑工具”，还是优先补“编辑器搜索 / 筛选 / 复制”效率工具
  - 战斗表现层是继续轻量优化，还是进入一轮更明确的演出 polish
