# 战斗执行器规格说明

## 1. 定位

战斗模块是 `battle` 类型事件的执行器，而不是整局主控制器。

世界层负责决定何时进入战斗、战斗结果如何影响单局。

## 2. 当前阶段范围

只做：

- 三类单位：敌人、友方、英雄
- 单位不可重叠
- 步行和飞行
- 近战和平射
- 基础追杀 AI
- 基础召唤 / buff 战斗内事件
- 基础胜负判定

不做：

- 抛射
- 墙体破坏
- 搜寻、巡航、保卫等复杂 AI
- 玩家拖拽选点施法

## 3. 输入输出

### 输入

世界层应传入 `BattleRequest`：

- `battle_id`
- `event_instance_id`
- `map_id`
- `hero_snapshot`
- `ally_snapshot`
- `equipped_relic_modifiers`

### 输出

战斗层应回传 `BattleResult`：

- `status`
- `victory`
- `defeat_reason`
- `casualties`
- `reward_package`
- `completed_objectives`
- `spawned_story_flags`
- `spawned_unlock_flags`
- `map_effects`

## 4. 子系统边界

### `BattleDirector`

- 驱动战斗生命周期

### `UnitController`

- 维护单位状态

### `AISystem`

- 处理索敌、追杀和攻击触发

### `CombatSystem`

- 处理伤害、攻击节奏和命中

### `BattleEventSystem`

- 处理战斗内事件的触发、通知和结束

## 5. 接口要求

- 战斗层不直接写永久存档
- 奖励只回传，不直接入库
- 世界层只读取结构化结果，不直接读战斗内部对象
