# 任务与叙事规格说明

## 1. 目标

该模块负责把“事件”包装成有目标、有影响、有推进感的任务体验。

任务不是事件的替代品，而是事件的叙事骨架。

## 2. 任务的职责

任务需要提供：

- 展示名称
- 任务目标
- 预期影响
- 奖励预览
- 出现条件
- 完成后推动的剧情与解锁

## 3. 任务与事件的关系

- `TaskDef` 负责“为什么出现、显示什么、完成后推进什么”
- `EventDef` 负责“它如何被处理”

推荐关系：

- 一个任务通常包装一个核心事件
- 某些任务完成后可以解锁后续固定事件
- 某些固定事件本身也可以是叙事任务节点

## 4. 固定叙事线

叙事事件大多数应属于固定事件线。

要求：

- 有明确进入条件
- 不被普通随机刷新覆盖
- 可由前置任务、剧情标记、key 物品或回合数触发
- 可在完成后生成新的固定事件
- 可要求玩家提交指定物品后开启下一段主线战斗

## 5. `story_flags` 与 `unlock_flags`

### `story_flags`

- 记录剧情推进
- 控制后续任务或固定事件出现

### `unlock_flags`

- 控制系统入口开放
- 控制 UI 页签开放
- 控制特殊事件或特殊物品入口出现

首批标准化开关：

- `narrative_events_enabled`
- `equipment_tab_enabled`
- `special_item_event_enabled`

## 6. MVP 叙事能力

当前阶段只做：

- 任务名
- 文本描述
- 1 到 3 个选项
- 每个选项对应结构化结果
- 支持“提交指定道具 -> 写入主线标记 -> 解锁主线战斗事件”

当前不做：

- 脚本语言
- 大量分镜演出
- 复杂对话树编辑器
