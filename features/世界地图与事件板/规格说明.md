# 世界地图与事件板规格说明

## 1. 目标

该模块负责把“单局搜打撤”转化为可见的地图回合事件板。

它需要回答：

- 当回合地图上出现哪些随机事件
- 当前有哪些固定事件仍然存在
- 玩家点击一个事件后，如何进入对应结算

## 2. 事件板组成

### 随机事件

- 每回合 `4-6` 个
- 通过随机池生成
- 刷新在地图固定锚点位置
- 下一回合可能被替换

### 固定事件

- 数量不定
- 来自任务线、叙事线、key 物品、保底逻辑
- 不占随机槽位
- 跨回合保留
- 在 A世界第2图样例中，主线叙事事件从第1回合开始保留

## 3. 关键规则

- 玩家每回合只处理 `1` 个事件
- 固定事件不能被随机刷新挤掉
- 随机事件和固定事件在 UI 上必须可区分
- 固定事件在完成、失效或被显式替换前持续存在
- 第1回合样例组合为：`3` 个战斗随机事件 + `1` 个随机事件 + `1` 个固定主线叙事事件
- 第2回合样例组合为：`5` 个随机事件 + `1` 个固定主线叙事事件
- 第4回合后允许触发额外强制战斗事件，该事件不占回合

## 4. 推荐运行时结构

### `TurnBoardState`

- `random_slots`
- `fixed_events`
- `selected_event_id`

### `RandomEventInstance`

- `instance_id`
- `event_id`
- `slot_id`
- `spawn_turn`

### `FixedEventInstance`

- `instance_id`
- `event_id`
- `line_id`
- `state`
- `spawn_turn`
- `expire_turn`

## 5. 生成流程

### `TurnStart`

1. 检查固定事件触发条件
2. 挂载新增固定事件
3. 从候选池中抽取 `4-6` 个随机事件
4. 将其放到地图随机锚点

### `PlayerChoose`

1. 玩家从地图上选择 `1` 个事件
2. 锁定选择，防止重复处理
3. 调用事件结算分发器

### `TurnEnd`

1. 回合数增加
2. 危险度增加
3. 保留有效固定事件
4. 重刷随机事件

## 6. 边界

该模块不负责：

- 战斗内部 AI
- 永久仓库存档
- 复杂剧情文本渲染

它只负责：

- 地图上的事件出现与保留
- 事件实例生命周期
- 将玩家选择转交给对应结算层
