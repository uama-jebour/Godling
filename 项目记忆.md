# 项目记忆

## 当前状态

- 当前仓库已经初始化 Godot 项目骨架、Autoload 和最小 JSON 数据
- 已确认的运行时原则是：Excel 只作为策划源，游戏运行时只读 JSON
- 已确认的事件模型是：每回合 `4-6` 个随机事件，加不定数固定事件
- 已确认的玩家样例流程是：家园配置 -> A世界第2图 -> 回合事件推进 -> 第4回合后可能强制袭击 -> 第6回合主线提交 -> 撤离事件
- 当前尚未进入正式功能实现阶段，文档是第一批可执行交付物

## 已确认的关键决策

- 顶层状态固定为 `ContentDB / ProgressionState / RunState`
- 战斗是事件执行器，不是整局唯一主控制器（包括普通战斗、强制战斗、撤离战）
- 叙事事件大多数属于固定事件线
- 固定事件不会被随机刷新覆盖
- 局内战利品和局外永久存档必须分层
- `story_flags` 和 `unlock_flags` 默认允许跨死亡保留
- 运行版本约束更新为 `Godot 4.6`

## 已知坑点

- 不要把“事件怎么出现”和“事件怎么结算”合并成一个枚举
- 不要默认用“三选一”理解回合事件板
- 不要把旧版长文档继续当成每次任务的完整输入
- 强制事件触发不占回合，不能被实现为普通随机事件替代

## 当前建议优先级

1. 巩固世界地图事件板，保证首回合组合和固定事件保留规则
2. 补完整事件出击面板与回合推进联动
3. 接入最小战斗执行器和强制袭击流程
4. 接入撤离事件和回家结算写入
